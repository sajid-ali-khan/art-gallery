# Art Gallery Portfolio

A minimal, elegant, dark-themed portfolio website for showcasing your art collection. Fully responsive, static, and deployable without any backend. Perfect for artists, illustrators, photographers, and creators.

---

## What This Is

This template provides:
- ✨ **Gallery-style portfolio** with responsive layout
- 🎨 **Organized collections** — group your work into categories
- 🔍 **Lightbox preview** — interactive image viewing
- ⚡ **Auto-generated catalogs** — Python script builds your image database
- 🚀 **Static deployment** — works on any hosting (GitHub Pages, Netlify, etc.)
- 🎯 **Minimal setup** — no databases, no backend required

---

## Getting Started

### Step 1: Clone the Repository

```bash
git clone https://github.com/YOUR_GITHUB_USERNAME/art-gallery.git
cd art-gallery
```

### Step 2: Personalize Your Portfolio

Edit `_config.yml` to change the title and description:

```yaml
title: Your Name — Your Art Title
description: A portfolio of your work.
```

Edit `index.html` to update your portfolio name in the title tag if desired.

### Step 3: Add Your Images

Replace the sample images with your own artwork:

1. **Delete sample images:**
   ```bash
   rm -rf images/book1 images/book2 images/random
   ```

2. **Create your own categories:**
   ```bash
   mkdir images/portraits      # or whatever your categories are
   mkdir images/commissions
   mkdir images/sketches
   ```

3. **Copy your images** into these folders:
   ```bash
   cp /path/to/your/artwork/* images/portraits/
   ```

**Supported formats:** `jpg`, `jpeg`, `png`, `webp`

### Step 4: Generate Your Data

Python automatically scans your image folders and creates a `data.json` file:

```bash
python generate.py
```

This creates your image catalog. You only need to run this after:
- Adding new images
- Reorganizing folders
- Adding new categories

### Step 5: (Optional) Add Titles & Descriptions

Open `data.json` and customize titles and captions for each piece:

```json
{
  "categories": [
    {
      "name": "portraits",
      "pieces": [
        {
          "file": "images/portraits/sarah.jpg",
          "name": "Sarah – 2024",
          "story": "Graphite on paper, 4 hours"
        }
      ]
    }
  ]
}
```

Both `name` and `story` are **optional** — leave them blank if you prefer.

### Step 6: View & Deploy

**View locally:**
- Open `index.html` in your browser

**Deploy to GitHub Pages:**
1. Push your repository to GitHub
2. Go to Settings → Pages → enable GitHub Pages for `main` branch
3. Your portfolio is live at `https://YOUR_USERNAME.github.io/art-gallery/`

**Deploy elsewhere:**
- Works on Netlify, Vercel, or any static hosting
- Just upload all files to your hosting provider

## How It Works

| File | Purpose |
|------|---------|
| `_config.yml` | Site title, description, and Jekyll config |
| `index.html` | Semantic markup for gallery & lightbox |
| `style.css` | Dark, moody aesthetic with responsive design |
| `script.js` | Fetches `data.json` and renders everything dynamically |
| `data.json` | Catalog of images (auto-generated by `generate.py`) |
| `generate.py` | Scans `images/` directory and creates `data.json` |
| `images/` | Your organized artwork folders |

## Common Workflows

### Add new pieces to an existing category

```bash
# 1. Copy images into category folder
cp my_new_portrait.jpg images/portraits/

# 2. Regenerate data
python generate.py

# 3. Refresh your browser
```

### Create a brand new category

```bash
# 1. Create folder
mkdir images/landscapes

# 2. Add your images
cp landscape01.png landscape02.png images/landscapes/

# 3. Regenerate
python generate.py

# The new category appears automatically!
```

### Edit titles and stories

```bash
# 1. Open data.json
nano data.json

# 2. Add or update "name" and "story" fields
# 3. Save file

# 4. Refresh browser (no regeneration needed)
```

## Customization Tips

### Change the color scheme
Edit `style.css` to modify the dark theme. Look for color variables near the top of the file.

### Add a bio or about section
Modify `index.html` to add your own content in the hero section.

### Organize by year or technique
Create categories like `images/2024/`, `images/watercolor/`, `images/digital/` — the script handles any folder structure.

### Use better image names
Instead of `IMG_123.jpg`, rename your images for clarity:
```bash
# Before
IMG_20260319_165316.jpg

# After
sarah-portrait-2024.jpg
```

Then run `python generate.py` again to update the catalog.

## Troubleshooting

**Images not showing?**
- Make sure images are in subdirectories under `images/` (not directly in `images/`)
- Run `python generate.py` again
- Check that filenames match exactly in `data.json`

**Python script not working?**
- Make sure Python 3 is installed: `python --version`
- Run from the project root directory
- Check file permissions: `chmod +x generate.py`

**Lightbox not working?**
- Make sure `script.js` and `style.css` are in the root directory
- Check browser console for errors (press F12)

## Design Philosophy

- **Dark & moody**: Charcoal tones (#0a0a0a, #1a1a1a) mirror graphite and paper
- **Typography-forward**: Cormorant Garamond serif for elegance, Poppins geometric sans for clarity
- **Subtle texture**: Grain overlay reinforces pencil medium
- **Smooth transitions**: Hover effects on images, fade/slide animations in lightbox
- **Gallery-like**: Unfolds like a personal exhibition, not a SaaS template
- **Fully responsive**: Mobile-first, adapts to all screen sizes

---

## Features

✓ Dynamic category tabs (from `data.json` — nothing hardcoded)  
✓ Masonry gallery grid with lazy loading  
✓ Lightbox with keyboard navigation (←→ or arrow keys)  
✓ Smooth image zoom & overlay title/story on hover  
✓ Preserves titles & captions across `generate.py` runs  
✓ Static HTML + CSS + JS (no build tools, no frameworks)  
✓ Deployable to GitHub Pages, AWS S3, Netlify, etc.

## Customization

### Change Artist Name & Subtitle

Edit `index.html`:
```html
<h1 class="artist-name">Your Name</h1>
<p class="artist-subtitle">Your tagline</p>
```

### Adjust Colors

Edit `style.css` variables (all hex/rgba in `:root` section or scattered throughout). Main palette:
- Background: `#0a0a0a`, `#1a1a1a`
- Text: `#f5f5f0`, `#e0e0e0`, `#a8a8a0`
- Accent: `rgba(200, 190, 180, ...)`

### Change Fonts

Edit import in `style.css`:
```css
@import url('https://fonts.googleapis.com/css2?family=YOUR_FONT_HERE&display=swap');
```

---

## Notes

- Images are **not optimized** by this script — use tools like ImageOptim or TinyPNG before uploading for faster loading
- `data.json` is generated fresh each time `generate.py` runs, but existing `name` and `story` fields are preserved
- The lightbox supports keyboard navigation: Esc (close), ← (prev), → (next)
- Dark mode only — no light theme toggle (intentional for this aesthetic)
